## 📘 TrueCode.Boo | 出口簿：團隊協作守則

歡迎加入這個數位自主實踐計畫。為了確保我們的內容能穩定地從這台「隨時會關機的筆電」與各位的終端發射到全世界，請遵守以下協作規範。

---

### 1. 寫作規範：檔案放哪裡？

所有的文章（Posts）都必須存放在：

`content/posts/檔案名稱.md`

- **檔名規範**：請使用小寫英文與連字號（例如：`radio-setup.md`），避免使用中文檔名，以防網址解析出錯。
    
- **圖片處理**：若有圖片，請放在 `static/images/` 資料夾，並在 Markdown 中使用 `/images/filename.jpg` 引用。
    

---

### 2. 必備的 Frontmatter (文章標頭)

每篇文章開頭必須包含以下 YAML 格式，否則 Hugo 無法渲染：

YAML

```
---
title: "這裡寫標題"
date: 2026-03-05
draft: false      # 記得檢查！如果是 true，網頁就不會出現
slug: "url-path"  # 強制指定網址，避免特殊字元造成 404
---
```

---

### 3. Git 提交三部曲

提交變更前，請務必先同步雲端狀態，避免「時空衝突」。

1. **拉取更新**（同步別人剛寫好的文章）：
    
    Bash
    
    ```
    git pull origin main
    ```
    
2. **本地檢查**（如果你的電腦有裝 Hugo，請先看一眼）：
    
    Bash
    
    ```
    hugo server
    ```
    
3. **推播發佈**：
    
    Bash
    
    ```
    git add .
    git commit -m "feat: 新增無線電專題文章"
    git push
    ```
    

---

### 4. 嚴禁事項 (極客紅線)

- **禁止修改 `.github/workflows/`**：除非你正在調整自動化部署流程。
    
- **慎動 `hugo.toml`**：這是基地的地基，若需增減選單或修改佈景，請先在群組溝通。
    
- **不要上傳大檔案**：圖片請先壓縮（單張建議 < 500KB），影片請上傳 YouTube 後使用 `{{< youtube ID >}}` 嵌入。
    

---

### 5. 常見問題：如果網頁沒更新？

1. 到 GitHub 倉庫的 **Actions** 頁面看有沒有「紅色叉叉」。
    
2. 檢查文章裡的 `draft` 是不是忘記改回 `false`。
    
3. 檢查 `hugo.toml` 裡的 `baseURL` 是否包含最後的斜線。
